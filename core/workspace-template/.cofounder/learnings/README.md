# learnings/

Accumulated context that should carry across sessions but doesn't belong in
the decisions log (which is for decisions, not general learnings) or in a
specific role's folder (because it applies broadly).

Examples of what goes here:
- Patterns the orchestrator noticed across multiple tasks ("Acme always
  wants a call before email — stop leading with email")
- Corrections the founder made more than once (a signal a skill should be
  updated — see Hermes's skill-curation loop, `cofounder-ideation.md` §9)
- Context about the company that isn't a one-time decision but ongoing
  operating knowledge (e.g. seasonal patterns, recurring blockers)

One file per learning topic, `.md`, named for the topic
(e.g. `customer-response-patterns.md`, `vendor-notes.md`). Keep entries
dated so stale learnings can be identified and pruned later.
